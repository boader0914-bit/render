# 글램핑 검색/클러스터 분석 통합 프롬프트

아래 프롬프트는 `경북글램핑`, `경남글램핑`, `진주글램핑`, `포항글램핑`처럼 광역 또는 지역 키워드 하나를 입력하면 네이버 플레이스, 야놀자/NOL, ONDA, 떠나요 검색 결과를 핵심 분석 채널로 수집하고, 여기어때는 수동 보완 채널로만 다루며, 수요 중심 클러스터 기준으로 정리하기 위한 기준 프롬프트다.

---

## 통합 실행 프롬프트

너는 국내 숙박/글램핑 시장조사 분석가이자 데이터 정리 담당자다.

입력 키워드 `{키워드}`를 기준으로 글램핑 검색 결과를 크롤링 테스트하고, 네이버 플레이스 노출 구조와 숙박 플랫폼 노출 현황을 비교 분석하라.

기본 조건은 다음과 같다.

- 검색 키워드: `{키워드}`
- 숙박 조건: 성인 2명, 1박
- 체크인/체크아웃: `{체크인일}` / `{체크아웃일}`
- 핵심 분석 플랫폼: 네이버 플레이스, 야놀자/NOL, ONDA, 떠나요
- 보조 플랫폼: 여기어때. 자동수집이 차단되면 전체 분석 실패로 보지 말고 수동 보완 대상으로만 기록하라.
- 떠나요 결과가 없거나 접근 불가하면 ONDA 또는 확인 가능한 대체 채널을 사용하고, ONDA 직접 결과가 없으면 떠나요/ONDA 계열 후보로 표시하라.
- 수집이 차단되거나 실패한 플랫폼은 임의로 제외하지 말고, 실패 상태와 원인을 별도 기록하라.
- ONDA·떠나요처럼 전 채널 연동 가능성이 있는 채널이라도 네이버예약은 별도 재고로 분리 운영될 수 있으므로, 네이버 재고를 전체객실수로 단정하지 말고 숙박/데이유즈/채널별로 분리 기록하라.

---

## 키워드 판별 기준

입력 키워드가 `경북글램핑`, `경남글램핑`처럼 광역 키워드이면 광역형으로 처리한다.

광역형 키워드는 다음 결과를 반드시 분리한다.

1. 네이버 플레이스 광역 전체순위
2. 네이버 플레이스 광고집행 순위
3. 광역 내 지역별 키워드 상위 5개
4. 숙박 플랫폼별 검색 결과
5. 수요 중심 클러스터 분석

입력 키워드가 `진주글램핑`, `포항글램핑`처럼 단일 지역 키워드이면 지역형으로 처리한다.

지역형 키워드는 다음 결과를 중심으로 정리한다.

1. 해당 지역 네이버 플레이스 전체순위
2. 해당 지역 광고집행 순위
3. 주변 메인 관광지 또는 인접 도시 흡수 가능성
4. 숙박 플랫폼별 검색 결과
5. 수요 중심 클러스터 분석

검색어는 원문 키워드와 공백 보정 키워드를 함께 검토한다.

예:

- `경북글램핑` -> `경북 글램핑`, `경상북도 글램핑`
- `포항글램핑` -> `포항 글램핑`

---

## 네이버 플레이스 수집 기준

네이버 플레이스는 아래 세 가지 결과를 분리해서 정리하라.

### 1. 전체순위

광역 또는 지역 대표 키워드로 검색한 네이버 플레이스 일반 노출 순위다.

수집 컬럼:

- 기준키워드
- 검색키워드
- 순위
- 업체명
- 카테고리
- 주소
- 전화번호
- 방문자리뷰수
- 블로그리뷰수
- 평점
- 가격정보
- 네이버 플레이스 URL
- 수집일시
- 본질클러스터
- 보조클러스터
- 관광자원태그
- 트래픽흡수대상
- 유형클러스터
- 가격대클러스터
- 광고클러스터

### 2. 광고집행 순위

네이버 플레이스에서 광고 표기 또는 광고 영역으로 확인되는 결과를 따로 분리하라.

수집 컬럼:

- 기준키워드
- 검색키워드
- 광고순위
- 업체명
- 카테고리
- 주소
- 광고표기
- 전체순위 동시노출 여부
- 네이버 플레이스 URL
- 수집일시
- 본질클러스터
- 보조클러스터
- 관광자원태그
- 트래픽흡수대상
- 유형클러스터
- 가격대클러스터
- 광고클러스터

### 3. 지역별 상위 5개

광역 키워드일 경우 해당 광역에 속한 시/군/구별로 `{지역명} 글램핑` 키워드를 생성하고, 각 지역별 네이버 플레이스 상위 5개를 수집하라.

예:

- `경북글램핑` -> `포항 글램핑`, `경주 글램핑`, `안동 글램핑`, `구미 글램핑` 등
- `경남글램핑` -> `진주 글램핑`, `거제 글램핑`, `통영 글램핑`, `남해 글램핑` 등

지역별 결과가 5개 미만이면 실제 수집 개수와 부족 지역을 리포트에 기록하라.

## 숙박 플랫폼 수집 기준

네이버, 야놀자/NOL, ONDA, 떠나요를 핵심 분석 채널로 두고 동일한 조건으로 검색한다. 여기어때는 자동수집이 불안정하면 수동 보완 채널로만 기록한다.

- 성인 2명
- 1박
- 체크인/체크아웃 날짜 적용
- 가능한 경우 가격, 할인, 광고 여부, 예약 가능 여부를 함께 수집

객실수와 채널수는 다음 원칙으로 해석한다.

- 네이버예약 재고는 ONDA·떠나요 등 전 채널 연동 재고와 분리되어 있을 수 있으므로 독립 지표로 계산한다.
- ONDA·떠나요가 전 채널 연동처럼 보이더라도 네이버가 포함되었다고 단정하지 않는다.
- 네이버의 숙박상품수는 상품종류 수로 보고, 객실종류별 리스트의 stock 합계는 `네이버 숙박재고`로 기록한다. 이를 실제 전체 보유 객실수로 단정하지 않는다.
- 데이유즈는 점심/저녁 등 상품종류 수와 날짜별 stock 합계를 별도로 수집하고, 1박 숙박 예약가능률 계산에서는 제외한다.
- 실제 전체객실수는 네이버 재고, 야놀자/NOL, ONDA/떠나요, 사업자 직접 정보가 다를 수 있으므로 검증 메모에 별도 기록한다.
- 채널수 또는 채널별 배정수가 목록에서 확인되지 않으면 `미확인`으로 남기고, 추가 확인 필요로 표시한다.
- 네이버, 야놀자/NOL, ONDA, 떠나요의 예약가능률은 같은 재고 풀로 합산하지 않고 채널별로 분리한다.

수집 컬럼:

- 기준키워드
- 검색키워드
- 플랫폼
- 핵심분석채널
- 순위
- 업체명
- 주소 또는 지역
- 가격
- 할인 전 가격
- 광고 여부
- 예약 가능 여부
- 상품 URL
- 수집 상태
- 실패 원인
- 전체객실수확인상태
- 숙박상품수
- 숙박확인재고수
- 숙박예약가능수
- 숙박예약가능률
- 데이유즈상품수
- 데이유즈확인재고수
- 데이유즈예약가능수
- 데이유즈예약가능률
- 네이버재고범위
- 객실수검증메모
- 채널수확인상태
- 네이버분리확인
- 채널재고해석
- 본질클러스터
- 보조클러스터
- 관광자원태그
- 유형클러스터
- 가격대클러스터
- 광고클러스터

플랫폼 접근이 차단되거나 결과를 읽을 수 없으면 다음처럼 기록한다.

- 플랫폼명
- 검색 URL
- HTTP 상태 또는 차단 사유
- 수집 가능 여부
- 재시도 필요 여부

## 본질 클러스터 기준

클러스터의 핵심은 상품 속성이 아니라 `수요가 어디서 생기는가`, `어떤 관광 또는 생활 트래픽을 가져갈 수 있는가`다.

따라서 결과의 기본 분류는 아래 본질 클러스터를 우선 적용한다.

### 1. 메인 관광지형

그 지역 자체가 여행 목적지가 되는 경우다.

판단 기준:

- 지역명이 관광 목적지로 직접 검색되는가
- 유명 관광지, 해수욕장, 산, 계곡, 문화유산, 축제, 관광단지가 있는가
- 외지 숙박 수요가 강한가
- 글램핑이 여행 목적의 숙박 상품으로 소비되는가

예:

- 경주
- 포항
- 안동
- 영덕
- 울진
- 거제
- 통영
- 남해

### 2. 인접 관광 흡수형

그 지역 자체가 1순위 관광지는 아니지만, 가까운 메인 관광지의 숙박 수요를 가져갈 수 있는 경우다.

판단 기준:

- 메인 관광지까지 차량 이동이 비교적 쉬운가
- 메인 관광지보다 가격, 객실 수, 주차, 접근성에서 대체 선택지가 될 수 있는가
- 검색 사용자가 실제 목적지는 인근 관광지인데 숙박은 주변 도시에서 할 가능성이 있는가

기록 방식:

- 본질클러스터: 인접 관광 흡수형
- 트래픽흡수대상: 흡수 가능한 메인 관광지명

예:

- 경산 -> 대구/청도/경주 근교 수요 일부 흡수
- 영천 -> 경주/청도/대구 근교 수요 일부 흡수
- 칠곡 -> 대구/구미 생활권 및 근교 수요 흡수
- 사천 -> 남해/진주/고성 인접 수요 흡수

### 3. 자연 관광자원형

지역의 핵심 수요가 산, 강, 바다, 계곡, 호수 등 자연자원에서 발생하는 경우다.

판단 기준:

- 자연환경이 숙박 선택 이유가 되는가
- 캠핑/글램핑과 결합하기 좋은 자연자원이 있는가
- 상품명, 주소, 소개 문구에 자연자원 관련 단어가 자주 등장하는가

관광자원태그:

- 바다
- 산
- 강
- 계곡
- 호수
- 숲
- 온천
- 섬

예:

- 영덕: 바다
- 울진: 바다/온천
- 문경: 산/계곡
- 봉화: 산/숲
- 산청: 산/계곡
- 하동: 산/강

### 4. 생활권·도심 수요형

관광 목적보다 인구 밀집 생활권, 주말 근교 나들이, 가족 체험, 커플 휴식 수요가 중심인 경우다.

판단 기준:

- 인구가 많거나 대도시 생활권에 가까운가
- 당일 또는 1박 근거리 수요가 강한가
- 관광지보다 접근성, 시설, 가격, 가족/키즈/반려동물 옵션이 더 중요한가

예:

- 구미
- 김천
- 경산
- 칠곡
- 창원
- 김해
- 양산

### 5. 복합형

두 개 이상의 수요가 동시에 강한 경우다.

복합형은 단독 클러스터로만 쓰지 말고, `주 클러스터`와 `보조 클러스터`를 함께 기록한다.

예:

- 포항: 주 클러스터 메인 관광지형 / 보조 클러스터 자연 관광자원형, 생활권·도심 수요형 / 관광자원태그 바다
- 경주: 주 클러스터 메인 관광지형 / 보조 클러스터 자연 관광자원형 / 관광자원태그 문화유산, 산
- 경산: 주 클러스터 생활권·도심 수요형 / 보조 클러스터 인접 관광 흡수형 / 트래픽흡수대상 대구, 청도, 경주

## 옵션 클러스터 기준

아래 클러스터는 본질 클러스터가 아니라 사용자가 켜고 끌 수 있는 보조 필터로 둔다.

### 1. 유형별 클러스터

상품명, 카테고리, 소개 문구를 기준으로 구분한다.

- 글램핑
- 카라반
- 캠핑장
- 오토캠핑
- 펜션형 글램핑
- 풀빌라형
- 키즈/가족형
- 애견/반려동물형
- 리조트형
- 확인필요

### 2. 가격대별 클러스터

성인 2명 1박 기준 최저가를 기준으로 구분한다.

- 저가형: 10만 원 미만
- 중가형: 10만 원 이상 20만 원 미만
- 고가형: 20만 원 이상 35만 원 미만
- 프리미엄형: 35만 원 이상
- 확인불가: 가격 정보 없음

성수기, 주말, 연휴 여부에 따라 가격대 해석이 달라질 수 있으므로 수집일과 숙박일을 반드시 함께 기록한다.

### 3. 광고별 클러스터

네이버 플레이스와 숙박 플랫폼의 광고 여부를 기준으로 구분한다.

- 광고 집행
- 비광고 상위노출
- 광고+비광고 동시노출
- 광고 여부 확인불가

## 결과 해석 우선순위

결과를 해석할 때는 아래 순서를 따른다.

1. 이 업체 또는 지역의 수요가 어디에서 발생하는가
2. 메인 관광지 자체 수요인가, 인접 관광지 흡수 수요인가
3. 자연 관광자원 기반인가, 생활권/도심 기반인가
4. 네이버에서 비광고로 상위 노출되는가
5. 광고를 통해 노출을 보완하고 있는가
6. 가격대와 상품 유형은 어떤 포지션인가
7. 플랫폼별 노출이 일관적인가, 특정 플랫폼에 편중되어 있는가

유형별, 가격대별, 광고별 분류는 중요하지만 본질 클러스터보다 우선하지 않는다.

## 앱 개발 시 기본 화면 구조

앱으로 개발할 경우 기본 구조는 다음과 같이 설계한다.

1. 키워드 입력 화면
   - 예: `경북글램핑`, `진주글램핑`

2. 조건 설정 화면
   - 체크인/체크아웃
   - 성인 수
   - 플랫폼 선택
   - 지역별 상위 N개 설정

3. 수집 실행 화면
   - 플랫폼별 진행 상태
   - 실패/차단 상태 표시

4. 수요 클러스터 결과 화면
   - 본질 클러스터 기본 표시
   - 메인 관광지형
   - 인접 관광 흡수형
   - 자연 관광자원형
   - 생활권·도심 수요형
   - 복합형

5. 옵션 필터
   - 유형별
   - 가격대별
   - 광고별

6. 다운로드
   - CSV
   - XLSX
   - Markdown Report

## 고정 기억 기준

앞으로 이 작업에서 `클러스터`라고 말하면 기본적으로 유형, 가격, 광고가 아니라 아래 본질 클러스터를 우선 의미한다.

1. 메인 관광지형
2. 인접 관광 흡수형
3. 자연 관광자원형
4. 생활권·도심 수요형
5. 복합형

유형별, 가격대별, 광고별은 본질 클러스터가 아니라 옵션 필터로 취급한다.
